# 平台变量文档 —— 形态示例

> 这是一份**示例**,用来说明"平台变量文档"长什么样、该怎么读。
> 用户的真实文档照此结构提供即可;实际可用的变量以用户提供的文档为准,不要照搬本文件里的变量当成一定存在。

平台在作业**执行期**会把 SQL 里的占位变量替换成实际值。变量写法形如 `${VAR_NAME}`,在 SQL 文本中按字符串原样保留即可,平台调度时再替换。

## 变量清单(示例)

| 变量 | 执行期替换为 | 典型用法 |
|---|---|---|
| `${TX_DATE}` | 作业实例执行日的**前一天**(`YYYY-MM-DD`) | 跑 T+1 增量:`WHERE tx_date = '${TX_DATE}'` |
| `${TX_DATE_NODASH}` | `${TX_DATE}` 的无横杠形式(`YYYYMMDD`) | 分区名拼接:`..._p${TX_DATE_NODASH}` |
| `${RUN_DATE}` | 作业实例执行当天(`YYYY-MM-DD`) | 记录加工时间 |
| `${MONTH_FIRST_DAY}` | 执行日所在月的 1 号(`YYYY-MM-01`) | 月初到当前的累计 |

## 使用约定(示例)

- 变量在 SQL 里**带单引号当字符串**用于日期比较:`WHERE stat_date = '${TX_DATE}'`。
- 增量装载典型形态:只处理 `${TX_DATE}` 当天的数据,而非全表全量。
- 变量是平台调度替换的,生成 SQL 时**原样保留** `${...}`,不要自己替换成具体日期。

## 给技能的提示

读到这类文档后:
- 当需求含"每天/每日跑""按交易日""增量""T+1"等语义时,优先用日期变量做增量过滤,而不是写死日期。
- 套用了哪个变量、含义是什么,要向用户说明并请其确认。
- 文档里没有的变量不要臆造。
